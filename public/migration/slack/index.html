Operating model

Important: Slack is the command and approval room. Odoo is the source of business records. Websites and subdomains bring intake. Agents process everything. Viewport receives only what agents cannot safely complete.
  1. Website/WhatsApp/form/client portal request arrives.
  2. Agent classifies it as MLG, MLH, or Viewport escalation.
  3. Agent creates or updates the correct Odoo record.
  4. Slack receives a short status/approval message.
  5. Human approves only when required.
  6. Agent continues execution and logs result back to Slack/Odoo.

Agents/Bots

Hermes/Modern Bot triage, draft, summarize, update internal records, and ask for approval. They do not bypass money/legal/security gates.

Viewport escalation

Only items agents cannot do safely go to Viewport: infrastructure, security, legal/accounting approval, production incidents, external API setup, DNS, paid-user decisions.

Agent control rules

Allowed

Agents can do

Read, summarize, classify, draft, create internal notes/tasks, prepare approval requests, update status, and route work.

Approval

Ask before

Invoices, payments, quotes sent to customers, contracts, purchase orders, tax/bank/payroll, permissions, customer messages.

Forbidden

Never automatic

Delete records, expose secrets, bypass approvals, mark paid without evidence, merge identities blindly, or mix MLG/MLH data lanes.
