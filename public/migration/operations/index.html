10 · Billion-$ Slack build → then Slack↔Odoo agents

Principle: Slack is the cockpit a construction group runs from on a phone. Gary taps buttons; Sam approves; agents do the work; Odoo records it; GitHub proves it.

  1. Secure the base — rotate leaked xoxb (#192), add MLG xapp (Socket Mode) + confirm signing secret; lock scopes to need.
  2. Slack bot service (services/agent-control-room, GitHub App + Slack Bolt) — fail-closed: signing, idempotency, retries, rate-limits, audit link per action.
  3. Approval Packet engine — universal primitive in #mlg-odoo-approvals / #mlh-*, buttons + expiry + escalation, audit in #mlg-odoo-audit.
  4. Odoo read path (L0/L1) — events → Slack posts (CRM lead, invoice, task, helpdesk ticket); drafts only.
  5. Business cockpit (Layer 3) — live-dashboard hourly refresh, div-* channels, living canvases; Telegram bridge for Gary.
  6. Slack→Odoo write path (L2→L3) — slash/buttons that, after approval, execute scoped Odoo writes via agents; never customer-facing/financial/legal without Sam/Dimitri.
  7. Bi-directional agent loop — Odoo automated actions → webhook → Slack; Slack approval → agent → Odoo; every step a GitHub task packet + evidence.
  8. Multi-company hard wall — MLG vs MLH companies/journals/channels; no cross-tenant.

Top-class markers: mobile-first cockpit, zero-spam (edit-in-place dashboards), every action reversible + audited, money/legal approval-gated, one memory brain, and a tenant template so the next client is a config — not a rebuild.
